# Release and PR

Purpose:
- coordinate prerelease, release-note, and PR-facing execution steps.

When to use:
- after developer validation is acceptable
- when the run is moving toward prerelease or PR publication

Prompt contract:
- use Jenkins MCP and GitHub release flow only through the frozen repo contracts
- do not improvise alternate release channels
- keep branch, tag, prerelease, and PR references traceable

Expected outputs:
- prerelease link
- PR link
- release-facing artifact summary
