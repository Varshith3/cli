# Ticket Intake Sufficiency

Purpose:
- decide whether the initiating ticket or user brief is detailed enough to plan safely.

When to use:
- at the start of a run before planning expands

Prompt contract:
- extract acceptance clues, blockers, and missing context
- ask only the minimum necessary clarifications

Expected outputs:
- `intake_summary`
- `clarification_questions`
- `acceptance_candidates`
