# PR Prerelease Commentary

Purpose:
- attach the current prerelease link to the PR in a reproducible way

When to use:
- after a prerelease exists
- during PR creation or PR reuse

Prompt contract:
- use the latest prerelease link already recorded by the run
- prefer the approved GitHub CLI path for adding the comment
- keep the comment compact, factual, and traceable

Expected outputs:
- `pr_prerelease_comment.md`
- `pr_prerelease_comment_result.json`
